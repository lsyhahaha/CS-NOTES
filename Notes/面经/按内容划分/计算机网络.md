# 面试常见题

### 1.什么是计算机网络？

计算机网络是指将多台计算机通过通信设备互连起来，实现信息和资源共享的系统。它是由一组通信协议、硬件设备和软件组成的，用于实现计算机之间的数据传输、通信和协作。计算机网络的目的是实现计算机之间的数据交换和通信，使得用户可以共享信息、资源和服务。通过计算机网络，用户可以远程访问其他计算机上的文件、数据和应用程序，进行远程协作和通信，以及访问互联网上的各种资源和服务。计算机网络的发展和普及极大地促进了信息技术的进步和社会的发展，成为现代社会不可或缺的基础设施之一。

### 2.计算机网络的分类有哪些？

计算机网络可以根据其覆盖范围、连接方式和使用场景等不同特点进行分类。以下是常见的计算机网络分类：

1. 局域网（Local Area Network，LAN）：覆盖范围较小，通常在办公室、学校等场所内部使用。局域网常用的技术包括以太网（Ethernet）和Wi-Fi等无线局域网技术。
2. 广域网（Wide Area Network，WAN）：覆盖范围较大，通常跨越多个地理区域，例如跨越城市、国家或大陆。广域网常用的技术包括传输介质如光纤、电缆以及协议如传输控制协议/互联网协议（TCP/IP）等。
3. 城域网（Metropolitan Area Network，MAN）：介于局域网和广域网之间，覆盖城市范围的网络，常用于连接多个局域网或办公场所。
4. 互联网（Internet）：由全球范围内的计算机网络互连而成，是全球最大的计算机网络。互联网连接了数十亿台设备，通过TCP/IP协议族进行通信。
5. 无线局域网（Wireless LAN，WLAN）：通过无线信号进行通信的局域网，常用于无线网络接入和移动设备的连接。
6. 蜂窝网络（Cellular Network）：基于移动通信技术，用于移动设备之间的通信，例如2G、3G、4G和5G网络。
7. 传感器网络（Sensor Network）：由大量分布在广域区域的传感器节点组成的网络，用于采集、处理和传输环境中的物理量和事件数据。
8. 对等网络（Peer-to-Peer Network）：没有中心服务器的网络结构，各个节点平等地协作和共享资源。

这些是常见的计算机网络分类，每种网络类型都有其特定的应用和使用场景，满足不同的需求和通信要求。

### 3.TCP/IP协议栈包括哪些层次

TCP/IP协议栈是计算机网络中最常用的协议栈，它包括以下四个层次（自下而上）：

1. 网络接口层（Network Interface Layer）：也称为数据链路层（Data Link Layer），负责将数据链路层协议封装成适合在物理网络上传输的帧。该层处理与物理介质的通信细节，例如以太网、Wi-Fi等。常见的协议有以太网协议（Ethernet）和无线局域网协议（Wi-Fi）。
2. 网络层（Internet Layer）：负责进行数据包的路由和寻址，确保数据在网络中正确地传输。最著名的协议是互联网协议（Internet Protocol，IP），它定义了数据在网络中的唯一寻址方式。其他常见的协议有互联网控制消息协议（Internet Control Message Protocol，ICMP）和互联网组管理协议（Internet Group Management Protocol，IGMP）。
3. 传输层（Transport Layer）：负责提供可靠的端到端数据传输服务。最常用的传输层协议是传输控制协议（Transmission Control Protocol，TCP），它提供面向连接的、可靠的数据传输。此外，用户数据报协议（User Datagram Protocol，UDP）是一种无连接的传输层协议，用于提供不可靠但高效的数据传输。
4. 应用层（Application Layer）：提供特定的网络应用服务和协议，允许用户访问网络上的资源。该层包括各种协议和服务，例如域名系统（Domain Name System，DNS）、超文本传输协议（Hypertext Transfer Protocol，HTTP）、文件传输协议（File Transfer Protocol，FTP）、邮件传输协议（Simple Mail Transfer Protocol，SMTP）等。

TCP/IP协议栈是互联网基础架构的核心，它提供了可靠的数据传输、寻址和路由等功能，支持广泛的应用和服务。

### 4.OSI参考模型有哪些层次？

OSI（开放式系统互联）参考模型是一个用于描述计算机网络体系结构的抽象框架，它由以下七个层次组成（自下而上）：

1. 物理层（Physical Layer）：负责在物理媒介上传输比特流，处理与传输媒介有关的电气、机械和功能特性。
2. 数据链路层（Data Link Layer）：提供可靠的点对点数据传输，通过帧（Frame）将比特流转换为数据块，并进行差错检测和纠正。
3. 网络层（Network Layer）：负责数据包的路由和转发，实现不同网络之间的互联和寻址。
4. 传输层（Transport Layer）：提供端到端的数据传输服务，负责数据的可靠传输、流量控制和拥塞控制。
5. 会话层（Session Layer）：建立、管理和终止应用程序之间的通信会话，提供会话控制和同步。
6. 表示层（Presentation Layer）：处理数据的表示方式，进行数据格式转换、数据加密和数据压缩等操作。
7. 应用层（Application Layer）：为用户提供网络应用服务，包括文件传输、电子邮件、远程登录和网络管理等。

OSI参考模型的设计旨在将计算机网络的功能划分为不同的层次，每个层次负责特定的任务，并通过定义接口与上下层进行通信。这种模块化的设计使得网络协议的开发、实现和管理更加灵活和可扩展。

### 5.TCP和UDP的区别是什么？

TCP（Transmission Control Protocol）和UDP（User Datagram Protocol）是两种常见的传输层协议，用于在计算机网络中提供数据传输服务。它们之间的主要区别如下：

1. 可靠性：
   - TCP：TCP是面向连接的协议，提供可靠的数据传输。它通过使用序号、确认和重传等机制来确保数据的可靠性和顺序性。
   - UDP：UDP是无连接的协议，不提供可靠性保证。它将数据报发送出去，但不关心是否到达目标主机，也不进行重传操作。
2. 传输方式：
   - TCP：TCP采用面向字节流的方式传输数据，会将数据分割成小的数据段进行传输，然后在接收端重新组装。它保证了数据的完整性，可靠性和顺序性。
   - UDP：UDP以数据报（Datagram）的形式传输数据，每个数据报都是独立的单元，没有拆分和重组的过程。每个数据报都有固定的大小，并保留了发送时的结构。
3. 连接性：
   - TCP：TCP是面向连接的协议，需要在通信双方建立连接后才能进行数据传输。它通过三次握手建立连接，四次挥手终止连接。
   - UDP：UDP是无连接的协议，发送端和接收端之间没有建立持久的连接，可以直接发送数据报。
4. 延迟：
   - TCP：由于TCP提供了可靠性保证，它在传输过程中会进行确认和重传操作，这些额外的处理会引入一定的延迟。
   - UDP：UDP不提供可靠性保证和重传机制，因此具有较低的延迟。
5. 适用场景：
   - TCP：TCP适用于对数据传输的可靠性要求较高的应用场景，例如文件传输、电子邮件、Web浏览等。
   - UDP：UDP适用于对实时性要求较高，但对数据可靠性要求相对较低的应用场景，例如音频、视频流传输、实时游戏等。

总体而言，TCP适用于要求可靠性和顺序性的数据传输，而UDP适用于实时性要求较高但可靠性要求较低的数据传输。选择使用哪种协议取决于具体的应用需求。

### 6.什么是IP地址？

IP地址（Internet Protocol Address）是用于在计算机网络中标识和定位设备的一种地址标识方式。它是一个由数字和点组成的标识符，用于唯一地标识网络中的每个设备（主机、路由器等）。IP地址用于在网络中进行数据传输和通信，类似于现实世界中的邮政地址。

IP地址由32位或128位二进制数表示，可以分为IPv4（32位）和IPv6（128位）两个主要版本。

IPv4地址是目前互联网上广泛使用的版本。它由四个8位二进制数（即四个字节）组成，每个数值范围在0到255之间，用点分十进制表示。例如，192.168.0.1就是一个IPv4地址。

IPv6地址是为了解决IPv4地址空间不足的问题而引入的新版本。IPv6采用128位地址，由8组16进制数字表示，每组数字之间用冒号分隔。例如，2001:0db8:85a3:0000:0000:8a2e:0370:7334就是一个IPv6地址。

IP地址的作用是定位和标识网络中的设备，使其能够在网络中进行通信和交换数据。它允许网络中的设备彼此发送和接收数据包，并根据目标IP地址将数据包正确地路由到目标设备。通过IP地址，我们可以访问远程服务器、发送电子邮件、浏览网页等各种网络活动。

### 7.IP地址的分类有哪些？

1. A类地址（Class A Address）：
   - 范围：1.0.0.0 到 126.0.0.0
   - 网络号占用：1个字节
   - 主机号占用：3个字节
   - 主机数量：约16,777,214个
   - 特点：适用于大型网络，如全球范围的组织或互联网服务提供商。
2. B类地址（Class B Address）：
   - 范围：128.0.0.0 到 191.255.0.0
   - 网络号占用：2个字节
   - 主机号占用：2个字节
   - 主机数量：约65,534个
   - 特点：适用于中等规模的网络，如大型企业或大学校园网络。
3. C类地址（Class C Address）：
   - 范围：192.0.0.0 到 223.255.255.0
   - 网络号占用：3个字节
   - 主机号占用：1个字节
   - 主机数量：约254个
   - 特点：适用于小型网络，如家庭网络或小型办公室网络。
4. D类地址（Class D Address）：
   - 范围：224.0.0.0 到 239.255.255.255
   - 用途：用于多播（Multicast）通信，可以将数据包同时发送给一组特定的接收者。
5. E类地址（Class E Address）：
   - 范围：240.0.0.0 到 255.255.255.255
   - 用途：保留地址，用于特殊目的，不用于常规的网络通信。

需要注意的是，上述分类方式是基于早期的IP地址分配方案，现在更常见的是使用无类别域间路由（Classless Inter-Domain Routing，CIDR）的方式进行IP地址的分配和管理。CIDR不再将IP地址严格划分为A、B、C等类别，而是使用前缀长度表示，如192.168.0.0/24表示一个带有24位网络前缀的网络。这种方式更灵活和高效，使得IP地址的分配更加合理和有效。

### 8.什么是子网掩码？

子网掩码（Subnet Mask）是用于划分网络中网络地址和主机地址的一个32位二进制数，与IP地址结合使用来确定网络地址和主机地址的范围。子网掩码指示了IP地址中哪些位用于表示网络地址，哪些位用于表示主机地址。

子网掩码的作用是将一个IP地址划分为网络地址和主机地址两部分，使得网络可以进行更细粒度的划分和管理。它通过对IP地址中的二进制位进行逻辑与操作来确定网络地址和主机地址。

子网掩码中的连续的1表示网络地址部分，连续的0表示主机地址部分。例如，对于IPv4的A类地址，其默认子网掩码为255.0.0.0，其中的前8位（8个连续的1）表示网络地址，后面的24位（24个连续的0）表示主机地址。

通过子网掩码，我们可以判断两个IP地址是否处于同一个子网中，以及确定网络地址的范围。例如，如果两个主机的IP地址与子网掩码进行逻辑与操作后得到的结果相同，那么它们就处于同一个子网中。

子网掩码的值决定了网络的划分粒度，较小的子网掩码表示更多的网络地址，较大的子网掩码表示更多的主机地址。通过合理选择子网掩码，可以灵活划分网络，满足不同规模和需求的网络环境。

### 9.解释ARP协议的作用


ARP（Address Resolution Protocol）是一种网络协议，用于将IP地址（网络层地址）解析为对应的物理MAC地址（数据链路层地址），从而实现在局域网中的IP地址到MAC地址的映射关系。

ARP协议的作用是通过局域网广播的方式，发送一个ARP请求包，询问某个IP地址对应的MAC地址。当网络中的主机收到ARP请求包后，如果发现自己的IP地址与请求中的目标IP地址匹配，就会返回一个ARP应答包，将自己的MAC地址告知请求方。

ARP协议的工作过程如下：

1. 当主机A要发送数据给主机B时，首先检查自己的ARP缓存中是否有主机B的MAC地址。如果有，则直接使用该MAC地址进行通信。
2. 如果主机A的ARP缓存中没有主机B的MAC地址，它就会发送一个ARP请求广播包，询问局域网上的所有主机，"谁的IP地址是主机B的IP地址？"。
3. 局域网上的其他主机收到ARP请求后，会检查自己的IP地址是否与请求中的目标IP地址匹配。如果匹配，就向主机A发送一个ARP应答包，将自己的MAC地址告知主机A。
4. 主机A收到ARP应答包后，将主机B的IP地址和MAC地址存储到ARP缓存中，并使用该MAC地址进行通信。

ARP协议的作用是在同一局域网中解析IP地址与MAC地址之间的映射关系，使得主机能够准确地发送数据到目标主机，从而实现网络通信。它是在数据链路层和网络层之间进行的地址解析，为上层协议（如IP协议）提供了必要的数据链路信息。

### 10.解释DNS的作用和原理

DNS（Domain Name System）是互联网中用于将域名转换为对应IP地址的系统。它是一个分布式的命名系统，提供了域名解析服务，使得用户可以通过易记的域名来访问互联网上的资源。

DNS的作用主要有两个方面：

1. 域名解析：DNS将人类可读的域名（如www.example.com）转换为计算机可理解的IP地址（如192.0.2.1）。当用户在浏览器中输入一个域名时，浏览器会向本地DNS服务器发送查询请求，本地DNS服务器会通过递归查询的方式向其他DNS服务器进行域名解析，最终找到对应的IP地址返回给浏览器，浏览器就可以根据该IP地址与目标服务器进行通信。
2. 域名管理：DNS还提供了域名管理的功能，允许域名所有者根据需要进行域名的注册、续费、转移等操作。域名管理涉及到域名的所有权、域名服务器的设置等内容。

DNS的工作原理如下：

1. 域名层次结构：域名被组织成一个层次结构，从右到左分为多个标签（Label）。顶级域名（Top-Level Domain，TLD）位于最右侧，例如.com、.org等。次级域名（Second-Level Domain，SLD）位于TLD的左侧，例如example.com。子域名（Subdomain）可以继续位于SLD的左侧，形成更深层次的域名结构。
2. DNS解析过程：
   - 本地域名解析：用户在浏览器中输入一个域名，首先会向本地DNS服务器发送查询请求。本地DNS服务器一般由互联网服务提供商（ISP）提供，通常存储了一部分常用域名的IP地址映射关系。如果本地DNS服务器缓存了请求的域名解析结果，则直接返回IP地址给用户。
   - 递归查询：如果本地DNS服务器没有缓存请求的域名解析结果，它会向其他DNS服务器发起递归查询。递归查询是指本地DNS服务器依次向根域名服务器、顶级域名服务器、次级域名服务器等进行查询，直到找到负责目标域名的权威域名服务器。
   - 迭代查询：当递归查询过程中的DNS服务器收到查询请求后，如果自身不是负责该域名的权威域名服务器，它会返回一个指向下一个级别的DNS服务器的地址给本地DNS服务器，让本地DNS服务器进行下一次查询，这个过程称为迭代查询。
   - 权威域名服务器：当本地DNS服务器最终查询到负责目标域名的权威域名服务器时，权威域名服务器会返回该域名对应的IP地址给本地DNS服务器，本地DNS服务器再将IP地址返回给用户的浏览器，完成域名解析过程。

通过DNS的工作，用户可以使用简单易记的域名来访问互联网资源，而无需记住复杂的IP地址。DNS的分布式架构和层次结构使得域名解析过程更高效和可靠。
